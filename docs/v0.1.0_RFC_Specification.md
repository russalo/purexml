# purexml v0.1.0 RFC — hardened safe-parse

> **Status: Approved** (2026-06-16, by Russell). Seed authored by the file-observer instance
> as the first-slice scope; the boundary cases below were refined in a cross-instance review
> round (purexml ⇄ file-observer, 2026-06-16 — see **Review history**) and approved in
> conversation. Stated as **capabilities + acceptance**, not implementation — implementation
> may now proceed against this spec, with a `docs/COMPLIANCE-v0.1.md` due before merge.

## 1. Scope of this slice

v0.1.0 delivers the **whole consumer contract** — capabilities **C1–C2** — because
file-observer's surface is a single hardened `fromstring`, and the value is only realized
when *both* "same parses succeed" and "same attacks blocked" hold together. This slice is
small in surface but **load-bearing in correctness**: it is a security control.

## 2. Capabilities delivered

- **C1 — Safe parse:** a `fromstring(text) -> Element` that returns a standard
  `xml.etree.ElementTree.Element` (consumer `.iter`/`.find` unchanged), stdlib-only.
- **C2 — Safe failure:** the blocked classes (§3) raise a **clear, catchable purexml
  exception** (a `ValueError` subclass — see §3.3); malformed input raises the stdlib
  `ParseError` unchanged. Either way: no entity expansion, no external fetch, no file read,
  no hang, no host-process crash.

## 3. The equivalence contract (the heart of this slice)

The design contract is **behavioral equivalence to `defusedxml.ElementTree.fromstring` at
its default configuration** — not a fresh hardening scheme, and not "maximally strict."
Matching the oracle's *allow* behavior is as much the contract as matching its *block*
behavior; over-blocking is a defect (it fails §4.1).

### 3.1 The pinned configuration

The oracle is `defusedxml.ElementTree.fromstring(text)` with defusedxml's defaults:

```
forbid_dtd      = False    # a DOCTYPE / internal DTD is ALLOWED
forbid_entities = True     # any entity DECLARATION is blocked
forbid_external = True     # blocks external reference RESOLUTION expat actually attempts
```

(Verified by reading the defusedxml 0.7.1 source + an empirical probe in §5 — see
`scratch/measure/v0.1.0_measure_first_findings.md`. The defaults above are exact.)

### 3.2 The block/allow matrix purexml must match

| Input | defusedxml default | purexml must | blocked by |
|---|---|---|---|
| Entity-expansion bomb (billion laughs / quadratic) — defines entities in the internal DTD | **block** (raises on the entity *declaration*, before expansion) | **block at declaration time** | `forbid_entities` |
| Internal general / parameter entity declaration (any) | **block** | **block at declaration** | `forbid_entities` |
| Declared external entity (XXE: `SYSTEM`/`PUBLIC` → local file or network) | **block** (at the entity *declaration* — **F3**: caught by `forbid_entities`, not external) | **block; no fetch, no read** | `forbid_entities` |
| External reference *resolution* expat actually attempts (e.g. a parameter entity pulling an external subset) | **block** | **block; no fetch** | `forbid_external` |
| **Unresolved external-DTD declaration** (`<!DOCTYPE r SYSTEM "…">`, never referenced) — **F2** | **allow** (parses; **no fetch occurs** — the handler never fires) | **allow** (do NOT over-block) | — |
| DOCTYPE / internal DTD with **no** entity declarations | **allow** | **allow** | — |
| Well-formed XML, no DTD | **allow** | **allow**, byte-identical tree | — |
| Malformed XML | **raise** stdlib `ParseError` | **raise** stdlib `ParseError` (pass through) | — |

> **F2 (measure-first correction).** Blocking external resources keys on *attempted
> resolution*, not on the *presence* of an external-DTD declaration. A `SYSTEM` DOCTYPE that
> is never referenced parses fine **and triggers no fetch** — empirically confirmed against
> the oracle. Raising on its mere presence would be over-blocking and would **fail §4.1**.

**Mechanism, not luck (finding #2).** Blocking MUST be proactive at the parser-handler
level — raise on the entity declaration *before* any expansion — exactly as defusedxml
does, **not** by relying on libexpat's billion-laughs amplification cap. The dev box runs
expat 2.6.1 (cap present, on by default), which would mask a non-equivalent parser; older
libexpat would not. Proactive blocking is what makes the result independent of the system
libexpat version.

### 3.3 Exception contract (finding #3)

"Behavioral equivalence" **cannot** include defusedxml's exception *classes* —
re-raising `defusedxml.EntitiesForbidden` would re-import the dependency purexml exists to
delete. Therefore:

- purexml ships **its own exception hierarchy whose blocked-input exceptions subclass
  `ValueError`** — mirroring defusedxml's `DefusedXmlException(ValueError)` MRO (**F4**), so a
  consumer that narrows to `except ValueError` stays equivalent. **Malformed** input raises
  the stdlib `xml.etree.ElementTree.ParseError` unchanged, exactly as the oracle does.
  (Confirmed against defusedxml 0.7.1 source — `DTDForbidden`/`EntitiesForbidden`/
  `ExternalReferenceForbidden` all subclass `ValueError`, not `ParseError`.)
- Equivalence is asserted at the **raises-vs-parses** level: for every input, purexml
  raises iff defusedxml raises, and parses (to an equivalent tree) iff defusedxml parses.
- The consumer migration is **swap the import**, not change the catch — file-observer's
  OOXML path funnels through one aliased `xml_fromstring` bound at import
  (`scanner.py:74-78`) and its structural-key site catches broadly (`except Exception`,
  `scanner.py:3193`), so the broad catch already tolerates purexml's type. If a consumer
  ever narrows its catch, the purexml exception type is the documented thing to catch.

## 4. Acceptance bar (binding)

1. **Same-parse equivalence vs `defusedxml`, compared via C14N (finding #4).** On the
   augmented corpus (§5), purexml's parse result MUST match `defusedxml`'s for every
   document that parses — **0 disagreements**. Equality is defined by canonicalization:
   `ET.canonicalize(ET.tostring(result, encoding="unicode"))` equals the oracle's. This
   removes attrib-order / tail-text / namespace-prefix ambiguity.
2. **Same-attack-blocked equivalence (falsify-first).** An adversarial battery — entity-
   expansion bombs, XXE targeting a local file, XXE targeting a network resource, and an
   external reference expat *attempts to resolve* (a parameter entity pulling in an external
   subset) — MUST be blocked by purexml exactly as `defusedxml` blocks each (raises;
   **provably** no expansion, no file read, no outbound request). The battery MUST also
   include the **allow** counter-cases (per F2: an unresolved external-DTD declaration; per
   §3.2: an entity-free internal DTD) and assert purexml does **not** over-block them.
   Written *before* the implementation is trusted.
3. **Proof harness, not assertion (finding #6).** "No fetch / no read" is proven, not
   assumed: during the attack battery, monkeypatch `socket.socket`, urllib's opener, and
   `builtins.open` to **raise** if called; point `SYSTEM` identifiers at a sentinel
   unreachable host and a sentinel local file; assert (a) the patched sink is never hit and
   (b) the sentinel's contents never appear in any parse output. A "no exception was raised"
   pass is not acceptable evidence.
4. **Zero runtime dependencies** — stdlib only (`defusedxml` is a dev/test oracle, never
   shipped, never imported under `src/`).
5. **Deterministic & never-crash-the-host (finding #2 scope).** Catchable errors only;
   bounded CPU/memory. Deterministic **for a given runtime, and independent of the system
   libexpat version** because blocking is proactive (§3.2).

## 5. Measure-first (first task)

Before the implementation is trusted:

- **Read the defusedxml source** and confirm: (a) the default config in §3.1; (b) the exact
  handlers that raise (entity-decl, unparsed-entity-decl, external-entity-ref, and the
  start-doctype handler) and on what; (c) its exception classes/hierarchy. Record the
  enumerated mitigation list — **it is the security contract this slice must match.**
- **Augment the corpus past OOXML (finding #5).** OOXML parts are DTD-free / entity-free,
  so a "0 disagreements" pass on them proves nothing about the allow-internal-DTD /
  forbid-entity-decls boundary that §3 turns on. Add boundary fixtures:
  - harmless internal DTD, no entities → **allow**;
  - internal entity declarations (general **and** parameter) → **block**;
  - namespaced tags, CDATA, comments, processing instructions → **allow**, tree-equal;
  - an encoding-declared prolog, including the `str`-with-encoding-declaration `ValueError`
    quirk — equivalence means matching that raise too.

## 6. Out of scope for v0.1.0

- **Configurable hardening flags (finding #7).** No `forbid_*` knobs. v0.1.0 hardcodes the
  defusedxml-default behavior — the consumer needs exactly one behavior, and a tunable
  security control is a misconfiguration surface. (Recorded in CLAUDE.md *Excluded
  decisions*.)
- Parse modes beyond `fromstring` (`iterparse`, SAX, `parse`-from-file) — add later only if
  a consumer needs them; file-observer does not.
- Any XML *writing* / serialization.

## 7. Runtime floor (measure-first note, not a blocker)

file-observer's `>=3.12` pin is conservative, not a hard floor (its real binding constraint
is `chardet`, ~3.10). `ET.canonicalize` is 3.8+ and **test-only**, so it does not constrain
purexml's runtime floor. Set purexml's floor wherever measure-first lands; going below 3.10
only requires confirming purexml's *own* runtime deps (it has none beyond stdlib), not the
consumer's. (`pyproject.toml` currently pins `>=3.12` — revisit.)

## 8. Definition of done

- C1–C2 implemented, stdlib-only, deterministic, blocking proactive at handler level.
- purexml exception hierarchy shipped (§3.3); consumer migration = import swap.
- Falsify-first suite: C14N same-parse sweep over the augmented corpus (0 disagreements vs
  `defusedxml`) **and** the adversarial attack battery with the no-fetch/no-read proof
  harness (each class blocked exactly as `defusedxml` does).
- The enumerated `defusedxml` mitigation list + default config recorded (the matched
  security contract).
- `HISTORY.md` entry; four-leg review run — **with extra weight on the adversarial leg**,
  since this is a security control (target spec §8).
