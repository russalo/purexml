# purexml v0.5.0 RFC — the trust surface

> **Status: Approved** (2026-06-16, by Russell; all §4 design questions resolved).
> The first minor under the *maintained-successor* mandate (`docs/v1.0_TARGET.md`):
> it doesn't add
> parse surface — it turns the vision's promises ("maintained / tracks the target",
> "the audit story is first-class") into a **shipped capability** + **visible
> evidence**. Stated as capabilities + acceptance + design questions, not implementation.

## 1. Scope of this slice
Two halves of one theme — *trust*:
- **(A) A public security-posture API** — the headline capability. A read-only call
  that tells an adopter what purexml protects them against **on their runtime**, what
  the libexpat layer covers, and what to opt into. This is the maintained-successor
  differentiator in code: the living library helps you understand your posture; the
  frozen incumbent never could.
- **(B) The audit story as shipped evidence** — make "proven, not asserted" visible:
  upgrade the differential-fuzz gate and emit a committed per-release equivalence
  report. Supporting, mostly test/CI.

Not more parse surface (minidom/sax stay demand-gated); not packaging (1.0).

## 2. Capability (A) — `purexml.security_report()`
A read-only introspection of the runtime's XML-security posture. No parse, no I/O
beyond reading `pyexpat` constants, no side effects, no behavior change to anything
else. Returns a **structured, printable `SecurityReport`** (a frozen object):

- `expat_version` — the runtime libexpat version tuple.
- `expat_meets_safe_floor` / `expat_meets_recommended` — bools vs the documented floors.
- `recommended_limits` — the `RECOMMENDED_LIMITS` preset.
- `mitigations` — per attack class, **where it's handled on this runtime**:
  - billion-laughs → blocked by purexml (entity-decl, default-on)
  - quadratic blowup → blocked by purexml (default-on)
  - XXE (external entity / external DTD retrieval) → blocked by purexml (default-on)
  - large-tokens / CVE-2023-52425 → mitigated by libexpat reparse-deferral **iff
    expat ≥ 2.6.0** (reports this runtime's status)
  - disproportionate dynamic memory → needs expat ≥ 2.7.2 (reports live/covered)
  - structural DoS (depth / attributes / size) → **opt-in** (covered only if the
    caller passes `Limits`; default-off)
- `notes` — human-readable warnings (e.g. "expat 2.6.1 < recommended 2.7.2: the
  disproportionate-memory class may be live; pass `RECOMMENDED_LIMITS` and/or upgrade").
- `__str__` → a readable multi-line posture summary, for logging/diagnostics.

Use: an adopter calls it at startup to log/gate ("is my runtime safe enough?"), or a
human runs `python -c "import purexml; print(purexml.security_report())"`.

## 3. Deliverable (B) — the audit evidence
- **Richer differential fuzz** vs the defusedxml oracle. Design choice (see §4): grow
  the seeded generator (no new dep) and/or add **coverage-guided fuzzing (Atheris)**
  as an optional `[fuzz]` dev-extra.
- **A committed equivalence report** — a generated artifact (counts + the
  0-disagreements result + the corpus manifest hash + expat/Python matrix) so "drop-in
  equivalence" is a *published fact per release*, not a claim. Lands at a stable path
  (e.g. `docs/EQUIVALENCE.md`), regenerated by the sweep.

## 4. Design decisions (resolved 2026-06-16)
- **`SecurityReport` shape: a structured, printable object** (frozen dataclass /
  namedtuple — typed fields + `__str__`), not a dict. **PROVISIONAL at 1.0** — it's
  novel defense-in-depth on a moving (libexpat) landscape, so the report's shape and
  contents may evolve in a minor (per ROADMAP freeze strategy); the *mirror* surface
  stays the frozen part.
- **`mitigations`: a mapping** keyed by a documented attack-class name, values a small
  enum (`BLOCKED` / `EXPAT_MITIGATED` / `OPT_IN` / `LIVE`).
- **Fuzz: both** — grow the seeded differential gate (no new dep, always-run in CI)
  AND add **Atheris coverage-guided** as an opt-in `[fuzz]` dev-extra (heavier
  tooling, deeper sweep on demand). Atheris is dev-only — never a runtime dep.
- **Naming: `security_report()`.**
- **Scope: confirmed** — posture API (A) + audit evidence (B) is the v0.5 slice.

## 5. Acceptance bar (binding)
1. **Zero impact on the mirror.** No parse-behavior change; no new *runtime* dep;
   the no-I/O guard still holds (`security_report` reads only `pyexpat`/stdlib).
   Default-everything stays byte-equivalent to defusedxml (sweep 0 disagreements).
2. **Report accuracy** — `security_report()` correctly reflects the runtime: verified
   against the actual `pyexpat.EXPAT_VERSION`, and the mitigation mapping matches the
   measured/enumerated reality (§6). Version-gated assertions (the report differs
   correctly on an expat below vs at the recommended floor — tested via the existing
   gating pattern).
3. **Deterministic + read-only** — same runtime → same report; no side effects; safe
   to call repeatedly / at import.
4. **Evidence runs in CI** — the (richer) differential fuzz is part of the matrix; the
   equivalence report regenerates deterministically from the sweep.

## 6. Measure-first (first task)
Enumerate the authoritative **attack-class → mitigation-layer** mapping (which class
is blocked by purexml's handlers, which by libexpat at which version, which is opt-in)
so `security_report()` is *accurate*, not decorative. Most is already established
(v0.1.2 + the `/deep-research` findings); formalize it as the data the report reads.

## 7. Out of scope for v0.5.0
- More parse surface — minidom/sax/pulldom (demand-gated). Packaging/PyPI/license
  (deferred to 1.0). The independent stdlib-internals-free parser (separate project).
- **Enforcing** the version assertion — `security_report()` *informs*; it does not
  change parse behavior or hard-fail. The enforce-vs-warn policy stays a 1.0 decision.

## 8. Definition of done
- `purexml.security_report()` → `SecurityReport` (read-only, printable), accurate per
  the §6 mapping; exported at top level. PROVISIONAL at 1.0.
- Richer differential fuzz in CI (+ optional `[fuzz]` Atheris extra if chosen);
  committed equivalence report regenerated by the sweep.
- No-I/O guard + default-off mirror intact. `HISTORY.md`; four-leg review;
  `COMPLIANCE-v0.5.md` before merge.
- Docs freshness sweep (per CONVENTIONS §3.2), incl. the ecosystem on-ramp note
  (the `security_report()` adopter story).
