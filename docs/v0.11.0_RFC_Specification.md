# purexml v0.11.0 RFC — map the reachable libexpat 2.8.2 classes

> **Status: Approved** (2026-06-26, by Russell — §3 resolved: aggregate class, key
> `integer_overflow_dos_expat_2_8_2`, retire `_HIGHEST_UNMAPPED_FIX`). A **report-only minor**
> (additive to `security_report().mitigations`; no parse-behavior change). The follow-up to the
> v0.10.1 floor patch (the v0.5.1→v0.6 lifecycle). Grounding:
> `scratch/review/v0.11.0_grounding_expat_2_8_2.md`.

## 1. Why
v0.10.1 bumped `RECOMMENDED_EXPAT_VERSION` to 2.8.2 and re-armed a generic floor advisory for the
2.8.2 integer-overflow / memory-corruption batch, leaving per-class mapping to this minor.
Grounding is done: **7 of the 2.8.2 CVEs reach purexml's ordinary parse paths**; the
reentrant-handler / suspend-resume trio (50219/56131/56412) and the xmlwf-only trio (56409–56411)
do **not**. Mapping the reachable batch makes the posture map complete for reachable classes again
and replaces the coarse advisory with a precise per-runtime status.

## 2. Scope (one slice)
Add one **aggregate** class to `security_report().mitigations`, gated on the 2.8.2 fix version, and
retire the interim generic advisory.

- New constant `_INTEGER_OVERFLOW_BATCH_FIXED = (2, 8, 2)`.
- New class **`integer_overflow_dos_expat_2_8_2`**: `EXPAT_MITIGATED if EXPAT_VERSION >=
  _INTEGER_OVERFLOW_BATCH_FIXED else LIVE`. An expat-layer integer-overflow / memory-corruption DoS
  batch (purexml can't block — below the Python layer — only report).
- A `notes` line (when LIVE) enumerating the 7 CVEs (storeAtts 56403 / addBinding 56404 /
  getAttributeId 56405 / XML_ParseBuffer 56406 / textLen 56407 / copyString 56408 / doProlog 56132).
- **Retire `_HIGHEST_UNMAPPED_FIX`** + its re-armed generic clause (every reachable fix is mapped
  again; the below-floor advisory now names the LIVE integer-overflow class).

## 3. Design decisions (resolved 2026-06-26)
1. **Aggregate** — one class, not 7 per-CVE rows (they share fix version + status + nature; the
   existing per-CVE classes only split because they had *different* fix versions). The CVEs are
   listed in the note.
2. **Key `integer_overflow_dos_expat_2_8_2`** — version-anchored (a batch, not one CVE).
3. **Retire `_HIGHEST_UNMAPPED_FIX`** (mirrors v0.9) — no reachable-unmapped fix remains.

## 4. Behavior / axes
- **No parse-behavior change.** Report-only; default mirror untouched; no-I/O guard holds.
- **SCHEMA** n/a. **LOGIC unchanged** — reported, not blocked. `mitigations` PROVISIONAL (additive).

## 5. Acceptance bar (binding)
1. No parse-behavior change; no new runtime dep; no-I/O guard holds; mirror byte-equivalent
   (empirical sweep — 0 divergences).
2. `integer_overflow_dos_expat_2_8_2` version-gated correctly (LIVE below 2.8.2, MITIGATED at/above);
   tested via monkeypatched `EXPAT_VERSION`.
3. The generic untracked-gap clause is gone (test: at 2.8.1 the advisory names the LIVE
   integer-overflow class and does NOT say "not yet individually tracked").
4. `security_report()` read-only/deterministic; immutability intact; `as_dict()` / `--json` pass the
   new key/status through.
5. Docs freshness sweep (CLAUDE Architecture + Known decision, HISTORY, SECURITY policy, ROADMAP);
   HISTORY row; four-leg (report-only tier); `COMPLIANCE-v0.11.md` before merge.

## 6. Out of scope
- The reentrant-handler / suspend-resume trio (50219/56131/56412) + the xmlwf trio (56409–56411) —
  grounded as not reachable; recorded, not mapped.
- `sax` (now v0.12.0; spec preserved at `scratch/v0.12.0_sax_RFC_approved.md`). No floor change
  (already 2.8.2); no packaging.
