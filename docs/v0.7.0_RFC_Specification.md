# purexml v0.7.0 RFC — the posture CLI: `python -m purexml`

> **Status: Approved** (2026-06-17, by Russell — all §6 questions resolved to the
> drafted leans). A minor: new public CLI surface over `security_report()`; no
> parse-behavior change. First build of the publish-worthy-debut push.

## 1. Why
The single best first-impression artifact for a cold evaluator: **one line that shows
them their own runtime's XML-security posture.** `security_report()` already produces
it; v0.7 makes it runnable without writing code — `python -m purexml` — so the README
demo is real, and an adopter gets a CI-gate + provenance hook for free. This is the
publish-worthy-debut gate (not FO; FO doesn't need it). New public surface → a minor.

## 2. Capability — `python -m purexml`
A `src/purexml/__main__.py` CLI over the existing `security_report()`:

- **`python -m purexml`** (no args) → the human-readable `security_report()` (its
  `__str__`). The 10-second demo. **Informs only — exit 0 always** (consistent with the
  inform-by-default version-assertion lean; the plain command never fails a build).
- **`python -m purexml --json`** → the report as machine-readable JSON (stable-ish,
  PROVISIONAL with the report). For logging / dep-provenance (scanner's `ScanContext`
  use case) and tooling.
- **`python -m purexml --check [--min-expat X.Y.Z]`** → **opt-in CI gate**: exit 0 if the
  runtime libexpat meets the floor, non-zero otherwise. Default floor =
  `RECOMMENDED_EXPAT_VERSION`; `--min-expat` lets a consumer **pin an explicit floor**
  (avoids the moving-default determinism break scanner flagged — they pass their pinned
  floor). Enforcement is the *caller* opting in, not the library — policy-consistent.
- **`python -m purexml --version`** → `purexml __version__` + libexpat version.

## 3. JSON shape (PROVISIONAL)
A `SecurityReport.as_dict()` (JSON-safe: version tuples → "x.y.z" strings, `mitigations`
→ plain dict, `recommended_limits` → `{max_depth,max_attributes,max_bytes}`, `notes` →
list) — reusable + unit-testable, so `--json` is `json.dumps(report.as_dict())`. Adds one
public method to the (already PROVISIONAL) `SecurityReport`.

## 4. The no-I/O guard (the real design tension)
`tests/test_no_io.py` scans **all** of `src/purexml/*.py`; a CLI legitimately needs
`argparse`/`json`/`sys` and writes stdout — which the strict library allowlist
(`xml`/`collections`/`types`) forbids. Resolution (design Q3): **`__main__.py` is the
one explicit I/O/CLI boundary** — exempt it from the strict *allowlist* check, but keep
it under the **FORBIDDEN** check (no `socket`/`urllib`/`http`/`subprocess`/`os`/…). So:
- the **library/parse surface** keeps its strict, unchanged guarantee (it still imports
  only `xml`/`collections`/`types`);
- the CLI may import only `argparse`/`json`/`sys` *on top* — none of which enable
  network/fs/exec — and the guard *proves* that, so the guarantee actually gets sharper
  (the one I/O boundary is named and bounded), not weaker.

## 5. Behavior / axes
No parse-or-block change; the library API is unchanged except the additive
`as_dict()`. **SCHEMA** n/a; **LOGIC** unchanged (the CLI reports, it doesn't parse
untrusted input). Feature-minor (new public surface) → full review tier per the rubric,
with leg 3 as the cheap mirror-proof (parse path untouched).

## 6. Design decisions (resolved 2026-06-17 — approved to the drafted leans)
1. **Surface:** `python -m purexml` via `__main__.py` for v0.7; the `purexml` console
   script deferred to publish-time packaging.
2. **Flag set:** default human report + `--json` + `--check [--min-expat]` + `--version`
   (all four — the demo doubles as a real adoption tool: provenance + opt-in CI gate).
3. **No-I/O guard:** `__main__.py` is the one I/O boundary — exempt from the strict
   library allowlist (may add `argparse`/`json`/`sys`) but STILL under the FORBIDDEN set.
4. **`as_dict()`** on `SecurityReport` (PROVISIONAL) backs the JSON path.
5. **Version:** v0.7.0, full minor flow → `COMPLIANCE-v0.7.md` before merge.

## 7. Acceptance bar (binding)
1. No parse-behavior change; default mirror byte-equivalent (sweep 0 disagreements).
2. `python -m purexml` prints the posture and **exits 0** (informs, never fails).
3. `--check` exit codes correct vs the floor (tested across monkeypatched expat
   versions, like `test_version_gating`); `--min-expat` pins correctly.
4. `--json` round-trips to the same data as the object; shape documented as PROVISIONAL.
5. No-I/O guard holds: parse surface strict; `__main__.py` adds only CLI-output stdlib,
   nothing forbidden — both asserted by `test_no_io`.
6. Read-only/deterministic; safe to run repeatedly.

## 8. Out of scope
- The `purexml` console-script name (publish-time). Any *parse* surface. Enforcing a
  floor by default (stays opt-in via `--check`; the enforce-vs-warn library policy is a
  1.0 decision). Colour/TTY formatting (keep output plain + pipeable).
