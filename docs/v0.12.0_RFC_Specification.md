# purexml v0.12.0 RFC — `purexml.sax` (+ `purexml.expatreader`) drop-in

> **Status: Approved** (2026-06-26, by Russell — §3 resolved: incl. expatreader; match
> defusedxml's `errorHandler` default; event-stream parity; proactive ref-cycle hygiene).
> A **feature minor** (new parse surface, security-load-bearing → full four-leg at depth).
> Next breadth module on the ratified/measured scope (sax = 375 sites). (Approved as v0.11.0
> on 2026-06-26; **renumbered to v0.12.0** because the libexpat-2.8.2 currency response took
> the v0.10.1/v0.11.0 slots.)

## 1. Why
sax is the #3 real consumer surface (375 sites) and the second breadth module after minidom.
It's a **different shape** — event-driven: a `ContentHandler` receives
`startElement`/`characters`/`endElement`/… callbacks from an `XMLReader`; nothing is
materialized into a tree. Legacy + streaming/framework code relies on it. With this, a sax
codebase migrates via `s/defusedxml/purexml/`.

## 2. Scope (one minor, two modules)
**A. `purexml.expatreader`** (Q1 = include; promotes expatreader deferred→done):
- `_DefusedExpatParser(xml.sax.expatreader.ExpatParser)` — installs the blocking handlers **in
  `reset()`** (where `ExpatParser` (re)creates `self._parser`), raising **purexml's** exceptions
  with the same arg-mapping as `_parser.py`/`minidom.py`.
- `create_parser(*args, **kwargs)` → `_DefusedExpatParser`.

**B. `purexml.sax`** — mirror `defusedxml.sax`:
- `make_parser(parser_list=[])` → hardened `XMLReader` (`parser_list` accepted + ignored, as
  defusedxml does — it always returns its own reader).
- `parse(source, handler, errorHandler=ErrorHandler(), forbid_dtd=False, forbid_entities=True, forbid_external=True)`
- `parseString(string, handler, errorHandler=ErrorHandler(), forbid_dtd=False, forbid_entities=True, forbid_external=True)`
  (with the `if errorHandler is None: errorHandler = ErrorHandler()` guard, mirroring defusedxml).
- defusedxml defaults (`forbid_dtd=False`, `forbid_entities=True`, `forbid_external=True`).

Malformed input surfaces as the stdlib `SAXParseException` via the `ErrorHandler` (equivalent to
the oracle), **not** a `PureXMLError`.

## 3. Design decisions (resolved 2026-06-26)
1. **Include `purexml.expatreader`** — the engine's natural home; near-free; closes the 14-site
   module. Conscious promotion expatreader *deferred→done*, recorded in the ROADMAP/Excluded ledgers.
2. **`errorHandler` default = match defusedxml** — literal `errorHandler=ErrorHandler()`
   (ruff is E/F-only, so a call-in-default is lint-clean; `ErrorHandler` is stateless, so the
   shared default instance is harmless). Strictest mirror; `parseString` keeps the `None`-guard.
3. **Equivalence = event-stream parity** — a recording `ContentHandler` captures the callback
   sequence (`startElement`/`endElement`/`characters`/`startPrefixMapping`/… as tuples) and asserts
   **equal to `defusedxml.sax`** over the real corpus; block-parity on the attack battery (same
   purexml exception types); malformed → `SAXParseException` on both; no-I/O trip-wire on the
   `parseString` attack path.
4. **Proactive reference-cycle hygiene** — `ExpatParser` holds `self._parser` with handlers bound
   to `self` (a cycle). Ensure the reader is refcount-reclaimable after parse/error up front
   (learned from minidom/PR#28); regression test.

## 4. Axes
- **New parse surface → LOGIC extended** (sax event path). Default behavior event-stream-equivalent
  to `defusedxml.sax`. **SCHEMA** n/a (drives the caller's stdlib handlers).
- **No new runtime dep** — `xml.sax` / `xml.sax.expatreader` are stdlib (`xml` package → permitted
  by the no-I/O import guard). Zero-dep intact.
- The sax + expatreader surface is **STABLE-track** at 1.0 (mirrors a frozen reference).

## 5. Acceptance bar (binding)
1. `make_parser`/`parse`/`parseString` (+ `expatreader.create_parser`) match `defusedxml.sax`:
   signatures + defaults; allow-path **event streams equal** to the oracle on a real corpus; attacks
   block with the **same exception types**; malformed → `SAXParseException` on both.
2. **No I/O on untrusted input** on the sax `parseString` path (socket/open/urlopen trip-wire).
3. `forbid_dtd=True` → `DTDForbidden`; the unparsed-entity + external-ref backstops fire.
4. Reader is refcount-reclaimable after parse/error (no lingering parser↔self cycle); regression test.
5. no-I/O structural guard passes (only `xml` stdlib under `src/`); mypy `--strict` + ruff clean;
   coverage ≥90% on the new modules.
6. Docs freshness sweep (COMPATIBILITY scope table, PUBLIC_CONTRACT/README/SECURITY/LIMITATIONS
   surface, ROADMAP: sax + expatreader done); HISTORY row; **full four-leg** (security-load-bearing);
   `COMPLIANCE-v0.12.md` before merge.

## 6. Out of scope (this minor)
- `xmlrpc` (distinct monkeypatch-the-stdlib shape — its own slice); `pulldom` (deferred).
  `lxml` excluded (zero-dep identity). No version-assertion policy change; no packaging.
