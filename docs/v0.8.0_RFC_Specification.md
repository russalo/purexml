# purexml v0.8.0 RFC — ship types: annotate the public surface + `py.typed`

> **Status: Approved** (2026-06-18, by Russell — §5 resolved to the drafted leans:
> mypy-clean + CI job, defer `--strict`; **preserve the verbatim mirror** in
> `_setevents` via `# type: ignore[misc]`; `mypy` in `[dev]` + a CI typecheck job;
> `types: mypy` badge; v0.8.0). A minor: ships types (new consumer-facing guarantee);
> no runtime/parse change.

## 1. Why
"Typed" was the one work-towards badge I **declined to fake** (the surface is unannotated;
`mypy src/` hits 6 errors). Earning it honestly is a real quality win for adopters: a
PEP 561 `py.typed` package means a consumer's type-checker uses purexml's *own* types
instead of falling back to `Any` — for a security control on the parse boundary, precise
types at the call site are genuine value. This is the last in-lane work-towards badge.

## 2. Capability
- **Annotate the public surface** so it type-checks meaningfully: the ElementTree family
  (`fromstring`/`parse`/`iterparse`/`fromstringlist`/`XML`/`XMLParser`/`tostring`/`ParseError`)
  + `security_report`/`SecurityReport`/`as_dict` + `Limits`/`RECOMMENDED_LIMITS` + the
  errors hierarchy + the expat-version API + `__main__.main`. Returns are concrete
  (`Element`, `ElementTree`, `Iterator[tuple[str, Any]]`, etc.).
- **Ship `py.typed`** (PEP 561 marker) so downstream type-checkers consume the types.
- **`mypy` clean in CI** — a typecheck job gates the claim (so the badge is enforced,
  like coverage/lint), and the README candidate gains a `types: mypy` badge.

## 3. The one real design wrinkle — `_parser._setevents`
`_setevents` is a **verbatim mirror of CPython's `XMLParser._setevents`** (a Known
decision: don't modernize away the mirroring — it's the equivalence guarantee). It
redefines a local `handler` per event branch with different signatures; mypy rejects that
("conditional function variants must have identical signatures", 6 errors). Options (Q2):
- **(a) `# type: ignore[misc]`** on each redefinition, with a comment that it's the
  deliberate stdlib mirror. *Lean — preserves byte-fidelity to CPython; a handful of
  documented, localized ignores.*
- (b) Rename each inner handler to a distinct local name. Cleaner types, no ignores, but
  cosmetically diverges from the verbatim mirror (behavior identical).

## 4. Behavior / axes
No runtime/parse behavior change — annotations + a marker file are inert at runtime.
`py.typed` IS a new consumer-facing guarantee (the package now ships types), so this is a
**minor** (new public surface). **SCHEMA** n/a; **LOGIC** unchanged. The public type
signatures become part of what freezes at 1.0 (note for the freeze checklist).

## 5. Design decisions (resolved 2026-06-18 — approved to the drafted leans)
1. **mypy bar:** clean on a sane config now (public surface fully annotated; internals
   typed enough to pass) + a CI `typecheck` job. `--strict` DEFERRED (possible later).
2. **`_setevents` pattern: preserve the verbatim mirror** — `# type: ignore[misc]` on the
   per-branch `handler` redefinitions, with a comment. (Don't rename / modernize the mirror.)
3. **`mypy` added to the `[dev]` extra + a CI typecheck job** (gates the claim).
4. **Badge:** `types: mypy`.
5. **Version: v0.8.0**, full minor flow → `COMPLIANCE-v0.8.md` before merge.

## 6. Acceptance bar (binding)
1. No runtime/parse-behavior change; default mirror byte-equivalent (sweep 0 disagreements);
   the no-I/O guard still holds (annotations add no imports beyond `typing`).
2. `mypy` clean (chosen config) on `src/purexml`, enforced by a CI job on the matrix's
   reference Python.
3. `py.typed` is **shipped in the wheel/sdist** (verified, not just present in the tree).
4. Public-API signatures are accurate (return types match what the functions actually
   return; no `Any`-washing of the public surface).
5. Suite green; honest `types: mypy` badge added, backed by the CI gate.

## 7. Out of scope
- `--strict` mypy (possible later). Annotating tests/ or fuzz/ (focus on `src/`).
- Any API/behavior change. The console-script, license, etc. (unrelated, deferred).
