## 1. Why

1.0 is a **governance declaration on a validated codebase**, not a feature release: the
public contract freezes, the backward-compat + deprecation policy turns on, and no new
features land at the freeze. Everything expensive-or-impossible to reverse afterward is
decided here. The build axis and the measured breadth surface are already complete; this
RFC is only about **what binds and how**.

The freeze exists because a stable dependency needs a promise a consumer can pin against —
file-observer's entire reason to swap is that `purexml.ElementTree.fromstring` (and the
exceptions around it) won't move under them after they pin.

## 2. What freezes vs. what stays provisional

Two tiers (CONVENTIONS §3.1 stability ladder):

- **STABLE at 1.0 — the `defusedxml`-mirror surface.** Its reference (`defusedxml`) is
  itself frozen (0.7.1 / 2021), so it is safe to lock hard: the module namespaces, the
  public function signatures, the exception hierarchy, and the mirror **defaults**.
- **PROVISIONAL at 1.0 — the novel defense-in-depth.** `Limits` / `RECOMMENDED_LIMITS`,
  `security_report()` / `SecurityReport` + its status vocabulary, and the libexpat
  version-assertion surface. These track the *moving* libexpat/CPython landscape, so
  "may change in a MINOR" lets them settle without a 2.0. They are present and usable at
  1.0 — just not under the hard contract.

## 3. The frozen (STABLE) surface

Enumerated so the freeze is exact. The freeze pins names + parameters + defaults + return
types. `tests/test_public_contract.py` guards this surface against silent drift.

### 3.1 Module namespaces (the `s/defusedxml/purexml/` contract)
`purexml.ElementTree` · `purexml.minidom` · `purexml.sax` · `purexml.expatreader` ·
`purexml.xmlrpc` · `purexml.common`, plus top-level convenience re-exports on `purexml`.

### 3.2 Functions + signatures (defaults are part of the contract)
- **ElementTree family** — `fromstring(text: str|bytes, forbid_dtd=False,
  forbid_entities=True, forbid_external=True, *, limits=None) -> Element`; `parse`,
  `fromstringlist`, `iterparse` (same `forbid_*` + keyword-only `limits`); `XMLParser`;
  the aliases `XML`, `XMLParse`/`XMLTreeBuilder`; re-exported stdlib `tostring`,
  `ParseError`.
- **minidom** — `parse(file, parser=None, bufsize=None, forbid_dtd=False, …, *, limits=None)
  -> Document`; `parseString(string: str|bytes, …) -> Document`.
- **sax** — `make_parser(parser_list=[], *, …)`; `parse(source, handler, …)`;
  `parseString(string: bytes, handler, …)` (**bytes-only**, mirrors defusedxml).
- **expatreader** — `create_parser()` / the public defused reader class.
- **xmlrpc** — `monkey_patch()`, `unmonkey_patch()`, `defused_gzip_decode(data, limit=None)`,
  `MAX_DATA`.
- **common** — `DefusedXmlException` (alias of `PureXMLError`) + the block-exception re-exports.

### 3.3 Exception hierarchy — COMPLETE (ratified)
```text
PureXMLError(ValueError)
├── DTDForbidden
├── EntitiesForbidden
├── ExternalReferenceForbidden
├── NotSupportedError
└── LimitExceeded            [PROVISIONAL — opt-in]
    ├── DepthExceeded
    ├── AttributesExceeded
    └── SizeExceeded
```
Malformed input raises the **stdlib** parser error for that surface —
`xml.etree.ElementTree.ParseError` (ElementTree), `xml.parsers.expat.ExpatError`
(minidom), `xml.sax.SAXParseException` (sax) — unchanged, never a `PureXMLError`.
`common.DefusedXmlException` is the frozen alias for catch-site migration. The set is
final: every reachable refusal is present; adding to this `except`-target hierarchy later
would be a 2.0-class change.

### 3.4 Defaults (frozen — mirror `defusedxml`)
`forbid_entities=True`, `forbid_external=True`, `forbid_dtd=False`. The default parse
path is byte-equivalent to `defusedxml`.

### 3.5 Runtime floor (frozen — FO contract point 6)
`requires-python >= 3.10` is a STABLE contract item: purexml will **not RAISE** its
supported Python floor above 3.10 within the 1.x line — a floor raise is a breaking
(2.0) change. (LOWERING the floor, e.g. to 3.9, is additive and allowed in a MINOR.)
Guarded by `test_fo_contract.py` (floor guard) + the CI 3.10–3.13 matrix.

### 3.6 `__version__` (frozen — FO contract point 5)
`purexml.__version__` is a plain `str`, always present, in the STABLE surface (consumers
stamp it into dependency-provenance / manifest checksums; a non-string/odd-repr broke a
consumer's determinism before). Guarded by `test_fo_contract.py`.

## 4. Ratified decisions (the one-way doors)

All resolved (Russell + FO, 2026-07-07); recorded here as the locked outcomes.

- **D1 — Import path. RATIFIED.** Name = `purexml` → the STABLE primary is the mirror
  namespaces `purexml.ElementTree.fromstring` (etc.), with top-level `purexml.fromstring`
  convenience also STABLE. FO pins `from purexml.ElementTree import fromstring` (ratified B2).
- **D2 — Exception-set completeness. RATIFIED.** The §3.3 set is COMPLETE + final.
- **D3 — Stability tiers. RATIFIED** as §2 — the mirror surface is STABLE; `Limits`, the
  posture report + its status vocabulary, and the version-assertion surface are PROVISIONAL.
- **D4 — Version-assertion policy. RATIFIED: inform-by-default, opt-in enforce; never
  auto-hard-fail on a moving floor** (a hard default gate would make a consumer's build
  environment-dependent). The **policy** is frozen, not the number (the floor moves:
  2.4.0 → … → 2.8.2 already).
- **D5 — Backward-compat + deprecation policy. RATIFIED (standard russalo shape):** MAJOR
  for breaking, MINOR additive-only, a symbol destined for removal marked deprecated ≥1 full
  MINOR before removal. On as of 1.0 (see `PUBLIC_CONTRACT.md`).
