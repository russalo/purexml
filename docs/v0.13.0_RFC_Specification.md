# purexml v0.13.0 RFC — `purexml.xmlrpc` (lazy-monkeypatch, option C)

> **Status: Approved** (2026-06-26, by Russell — §3.1 = **option C**: lazy-import monkeypatch so
> the import-time no-I/O guarantee stays intact). A **feature minor** (new surface — a
> transport-hardening shim, not a parse fn; security-load-bearing → full four-leg). The last
> non-negligible measured module (`defusedxml.xmlrpc`, 343 sites). Grounding +
> identity analysis: `scratch/v0.13.0_RFC_DRAFT.md`.

## 1. Why / shape
`defusedxml.xmlrpc` is **not a parse function** — it **monkeypatches the stdlib `xmlrpc`** to make
`xmlrpc.client` parsing + gzip handling safe: it swaps in a defused expat parser (`FastParser`),
bounds gzip decompression (`MAX_DATA = 30 MB`, anti-decompression-bomb), and patches
`gzip_decode`/`GzipDecodedResponse` on `xmlrpc.client`/`xmlrpc.server`. It is the only measured
module that (a) mutates global stdlib state and (b) touches network-capable stdlib (`xmlrpc.client`
→ `http`/`socket`) + `gzip`.

## 2. The identity decision (resolved: option C)
purexml's pitch is a **pure, no-I/O-at-import parse surface**. xmlrpc conflicts with that. **Option
C keeps the conflict contained:** all `xmlrpc`/`gzip`/`io` imports are **LAZY** — performed *inside*
`monkey_patch()` and the gzip helpers, never at module top. So **`import purexml` (and
`import purexml.xmlrpc`) never pulls `xmlrpc.client`/`socket`/`http`/`gzip`** — the network-capable
import happens only when a consumer *explicitly* calls `monkey_patch()`. The structural no-I/O
guarantee at import time is preserved; the opt-in patch is where the network-capable code enters.

## 3. Scope (mirror `defusedxml.xmlrpc`, lazily)
- `purexml.xmlrpc.monkey_patch()` / `unmonkey_patch()` — patch/restore `xmlrpc.client.FastParser`,
  `.GzipDecodedResponse`, `.gzip_decode` (+ `xmlrpc.server.gzip_decode`). Originals captured on
  first `monkey_patch()` for restore.
- A **lazy class factory** builds + caches `DefusedExpatParser(xmlrpc.client.ExpatParser)` (the
  defused parser with purexml's forbid_* handlers, arg-mapped) — it can't be a module-level class
  (that would import `xmlrpc.client` at import time), so it's defined on first patch.
- **Gzip-bomb defense:** `MAX_DATA = 30 * 1024 * 1024`; `defused_gzip_decode(data, limit=None)`
  (lazy `gzip`/`io`, bounds decoded size); a lazy `DefusedGzipDecodedResponse(gzip.GzipFile)`
  factory. purexml's first non-XML defense (decompression bomb).
- `defuse_stdlib()` (defusedxml's cross-module patcher) — **out of scope** (separate concern).

## 4. The no-I/O guard (the load-bearing part)
- **Static guard carve-out:** `tests/test_no_io.py` walks ALL imports (`ast.walk`), so the lazy
  `xmlrpc`/`gzip` imports are still seen statically → add an `XMLRPC_EXTRA = {"xmlrpc", "gzip", "io"}`
  carve-out for `xmlrpc.py` (like `CLI_EXTRA` for `__main__.py`), with the justification that they
  are lazy + opt-in. **`xmlrpc.py` must NOT appear to import `socket`/`http`/`urllib` directly**
  (those stay FORBIDDEN even for `xmlrpc.py` — purexml never imports the transport itself; the
  network reach is `xmlrpc.client`'s, the consumer's, only after `monkey_patch()`).
- **Behavioral lazy test (the real guarantee):** a subprocess test asserting that after
  `import purexml.xmlrpc` (fresh interpreter), `socket` / `http.client` / `xmlrpc.client` are NOT in
  `sys.modules` — and ARE only after `monkey_patch()` is called. This is what makes option C honest.

## 5. Axes
- New surface (a monkeypatch shim). **LOGIC extended** (gzip-bomb + xmlrpc parser hardening).
  **SCHEMA** n/a. Zero-dep intact (all stdlib); the no-I/O *import* guarantee is preserved by laziness.

## 6. Acceptance bar (binding)
1. `monkey_patch()` installs the defused parser + bounded gzip on `xmlrpc.client` (+ server
   gzip); `unmonkey_patch()` restores. Verified by checking the patched attributes + a round-trip.
2. The defused xmlrpc parser blocks entity/DTD/external attacks with purexml's exceptions; allow
   inputs parse equivalent to `defusedxml.xmlrpc`'s defused parser.
3. **gzip-bomb defense:** a small payload that decompresses beyond `MAX_DATA` raises `ValueError`
   ("max … length exceeded"); a normal payload round-trips. Parity with `defused_gzip_decode`.
4. **Import-time no-I/O preserved:** the behavioral subprocess test (`socket`/`http`/`xmlrpc.client`
   absent from `sys.modules` after `import purexml.xmlrpc`, present after `monkey_patch()`).
5. Static no-I/O guard passes (xmlrpc.py carve-out; `socket`/`http`/`urllib` still forbidden there);
   mypy `--strict` + ruff clean; coverage ≥90%.
6. Docs freshness sweep; HISTORY row; **full four-leg** (security-load-bearing); `COMPLIANCE-v0.13.md`.

## 7. Out of scope
- `defuse_stdlib()`; `pulldom` (deferred); `lxml` (excluded). No packaging.
