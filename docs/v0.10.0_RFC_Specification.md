# purexml v0.10.0 RFC — minidom drop-in + `purexml.common` compat shim

> **Status: Approved** (2026-06-19, by Russell — §3 resolved to the drafted leans). A
> **feature minor** (new parse surface, security-load-bearing → full four-leg at depth).
> Scope grounded in `scratch/research/2026-06-19_defusedxml-usage-measurement.md` (ratified
> 2026-06-19): minidom (457 sites) + sax (375) are the load-bearing breadth that turns
> purexml from a one-adopter ElementTree slice into an ecosystem-adoptable defusedxml successor.

## 1. Why
purexml deserves a 1.0 only if a *stranger* can adopt it — not just the ElementTree slice
file-observer needs. `defusedxml.minidom` (DOM) is the #2 real consumer surface and is
**security-load-bearing** (DOM parsing carries the same XXE / entity-expansion / external-DTD
attack surface). This is the first real breadth: a minidom codebase can now `s/defusedxml/purexml/`.
It also lands `purexml.common` (419 import sites) — the exception-compat shim that makes
catch-sites migrate verbatim across **every** module.

## 2. Scope (one minor, two pieces)
**A. `purexml.minidom`** — mirror `defusedxml.minidom`:
- `parseString(string, parser=None, forbid_dtd=False, forbid_entities=True, forbid_external=True)` → `xml.dom.minidom.Document`
- `parse(file, parser=None, bufsize=None, forbid_dtd=False, forbid_entities=True, forbid_external=True)` → `Document`
- defusedxml defaults (`forbid_dtd=False`, `forbid_entities=True`, `forbid_external=True`).
- **Mechanism (mirror):** subclass stdlib `xml.dom.expatbuilder.ExpatBuilder` (+ `Namespaces` →
  `DefusedExpatBuilderNS`) and `install()` blocking handlers that raise **purexml's** exceptions,
  mapping expat's callback args to purexml's constructors **exactly as `_parser.py` does**
  (`EntitiesForbidden(name, sysid, pubid)` etc. — purexml's signatures differ from defusedxml's,
  so handler bodies map args, they are not copied verbatim). Returns a **standard stdlib `Document`**.

**B. `purexml.common`** — exception-compat shim: re-export purexml's `EntitiesForbidden` /
`ExternalReferenceForbidden` / `DTDForbidden` / `NotSupportedError` (names already match defusedxml)
**+ alias `DefusedXmlException = PureXMLError`** (only base-name difference; same `ValueError` MRO),
so `from defusedxml.common import DefusedXmlException; except DefusedXmlException` survives
`s/defusedxml/purexml/`. (`NotSupportedError` is added to `errors.py` as `NotSupportedError(PureXMLError)`.)

## 3. Design decisions (resolved 2026-06-19 — approved to the leans)
1. **Custom `parser=` / `bufsize=`: option (b), security-first.** The default path
   (`parser is None and not bufsize`) is the fully-hardened expatbuilder. A custom `parser` →
   **`NotSupportedError`** (a foreign parser bypasses purexml's hardening — a footgun a security
   library must refuse, not silently honor). `bufsize` is accepted for signature-compat and does
   not alter the hardened whole-document parse (the resulting `Document` is identical; documented).
   *(Rationale: defusedxml routes these to a pulldom path; purexml has no pulldom yet, and shipping
   a half-hardened fallback is worse than a clear, recorded refusal. Custom-parser usage is
   measured-negligible.)*
2. **`purexml.common` contents:** the 4 names + `NotSupportedError`. The compat names live at
   `purexml.common` ONLY; the top-level `purexml` namespace stays purexml-native (no
   `DefusedXmlException` at top level).
3. **DOM equivalence oracle:** "same parse" = produced `Document` C14N-equal to
   `defusedxml.minidom` via `ET.canonicalize(doc.toxml())` where well-formed, plus a structural
   node-walk on a real corpus; block-parity on the attack battery (same exception types); no-I/O
   trip-wire on the parseString attack path.
4. **Layout:** `src/purexml/minidom.py` + `src/purexml/common.py`; eagerly imported from
   `__init__` (parity with `ElementTree`); `purexml.minidom` mirrors `defusedxml.minidom`.

## 4. Axes
- **New parse surface → LOGIC extended** (adds the minidom DOM path; the mitigation set now covers
  minidom's entry points). Default behavior on the path is structure-equivalent to defusedxml.minidom.
  **SCHEMA** n/a (returns stdlib `Document`).
- **No new runtime dep** — `xml.dom.minidom` / `xml.dom.expatbuilder` are stdlib (`xml` package →
  permitted by the no-I/O import guard's `ALLOWED_TOPLEVEL`). Zero-dep intact.
- The minidom + common surface is **STABLE-track** at 1.0 (mirrors a frozen reference, like ElementTree).

## 5. Acceptance bar (binding)
1. `parseString`/`parse` match `defusedxml.minidom`: same signature + defaults; allow-path Documents
   C14N/structure-equivalent to the oracle on a real corpus; attacks block with the **same exception
   types** (`EntitiesForbidden`/`ExternalReferenceForbidden`/`DTDForbidden`); malformed → stdlib error.
2. **No I/O on untrusted input** — socket/open/urlopen trip-wire holds on the minidom parseString path
   (new parse surface = where the no-fetch bug re-emerges; explicitly tested).
3. `purexml.common` provides the defusedxml exception names; `except DefusedXmlException` catches a
   purexml block; custom `parser` raises `NotSupportedError`.
4. no-I/O structural guard still passes (only `xml` stdlib imported under `src/`); mypy `--strict`
   + ruff clean; coverage ≥90%.
5. Docs freshness sweep (COMPATIBILITY scope table, PUBLIC_CONTRACT surface, ROADMAP scope promotion,
   Excluded-ledger `lxml` entry, CLAUDE Architecture); HISTORY row; **full four-leg** (feature-minor,
   security-load-bearing); `COMPLIANCE-v0.10.md` before merge.

## 6. Ledger decisions ratified 2026-06-19 (recorded on merge)
- **ROADMAP:** `minidom` + `sax` promoted *deferred → in earned 1.0 scope* (evidence-backed).
- **Excluded ledger:** `lxml` — **excluded on zero-dep identity** (wraps third-party `lxml`;
  deprecated upstream). 390 sites exist; the exclusion is principled, not demand-based.

## 7. Out of scope (this minor)
- `sax` (next breadth minor), `xmlrpc` (separate monkeypatch-shape slice), pulldom/expatreader as
  consumer-facing modules (expatbuilder is the internal minidom engine, not exposed). lxml (excluded).
  No version-assertion policy change; no packaging.
