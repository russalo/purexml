# purexml v0.2.0 RFC — complete the non-streaming ElementTree surface

> **Status: Approved** (2026-06-16, by Russell; namespace design endorsed by the
> anchor consumer, file-observer — see Review history). Scope set by the ratified
> `docs/ROADMAP-to-1.0.md` (G3, v0.2). Stated as capabilities + acceptance, not
> implementation — implementation may proceed against this spec, with a
> `docs/COMPLIANCE-v0.2.md` due before merge.

## 1. Scope of this slice
v0.2.0 makes purexml a **drop-in for the *non-streaming* `defusedxml.ElementTree`
surface** — every parse entry point except `iterparse`, plus the parameterized
`forbid_*` signature. It rides directly on the proven v0.1 `_HardenedParser`
(expat + TreeBuilder + blocking handlers). **`iterparse` is explicitly deferred to
v0.3** — incremental hardening is the one risk-bearing piece and earns its own RFC.

## 2. Capabilities delivered
Match `defusedxml.ElementTree`'s API and behavior for:
- **`parse(source, parser=None, *, forbid_dtd=False, forbid_entities=True,
  forbid_external=True)`** — parse from a filename or file-like object → hardened
  `ElementTree`. (defusedxml returns an `ElementTree`; mirror that, incl. `.getroot()`.)
- **`fromstringlist(sequence, ...)`** — parse a sequence of string fragments.
- **`XML`** — alias for `fromstring` (defusedxml exports it).
- **`tostring`** — re-export of stdlib `xml.etree.ElementTree.tostring` (serialization;
  no hardening — defusedxml re-exports it for API completeness).
- **Expose the hardened parser class** (defusedxml's `DefusedXMLParser` /
  `XMLParser` / `XMLTreeBuilder` aliases) for callers passing `parser=`.
- **The `forbid_*` knobs across the whole surface** (re-opened 2026-06-16) — same
  parameter names, order, and defaults as defusedxml. This requires:
  - a new **`DTDForbidden`** exception (a `ValueError` subclass, mirroring
    defusedxml) — *reachable only when `forbid_dtd=True`*;
  - wiring `StartDoctypeDeclHandler` → `DTDForbidden` when `forbid_dtd=True`;
  - parameterizing handler installation in `_HardenedParser`.

## 3. Module layout — `purexml.ElementTree` canonical (endorsed by the anchor consumer)
Introduce **`purexml.ElementTree`** as the canonical namespace mirroring
`defusedxml.ElementTree`. For the implemented surface the migration is a literal
`s/defusedxml/purexml/` with **zero structural change** — confirmed against
file-observer's real source (`from defusedxml.ElementTree import fromstring as
xml_fromstring` → `from purexml.ElementTree import …`). After v0.2:
```python
from purexml.ElementTree import fromstring, parse, fromstringlist, XML, XMLParser, tostring, ParseError
# iterparse arrives in v0.3
```
Keep top-level convenience re-exports (`purexml.fromstring`, the exceptions,
`__version__`) for new consumers without the defusedxml lineage. Deciding the
layout *now*, while we add the family, pre-empts the A1 freeze trap (import path is
frozen contract) and lets adoption validate it.

**Scope honesty (do not over-promise the sed):** the `s/defusedxml/purexml/`
guarantee holds **only for the subset purexml implements.** A consumer using a
defusedxml name purexml hasn't shipped yet (e.g. `defusedxml.ElementTree.iterparse`
before v0.3) gets a textually-successful sed that then raises **`AttributeError`**
on access — a loud, immediate failure, never a silent wrong-parse. The RFC promises:
*"`purexml.ElementTree` mirrors `defusedxml.ElementTree`'s shape for the implemented
surface; unimplemented names are absent."*

**Re-export decision (faithful mirror):** `purexml.ElementTree` re-exports exactly
the stdlib names `defusedxml.ElementTree` does — **`ParseError` and `tostring`** (both
in defusedxml's `__all__`) — so `from purexml.ElementTree import ParseError` works
identically. Note: faithful = *match defusedxml*, and defusedxml.ElementTree does
**not** export `Element` (it's not in its `__all__`/imports), so purexml won't add
it either; consumers needing `Element` import it from stdlib `xml.etree.ElementTree`,
exactly as they would with defusedxml. file-observer catches broadly and needs none
of these, but the faithful mirror serves consumers who do import them.

## 4. The equivalence contract (extends v0.1 §3)
Each new entry point MUST be behaviorally equivalent to its `defusedxml.ElementTree`
counterpart, at every `forbid_*` combination:
- **same-parse:** C14N-equal element tree for documents that parse (`parse`,
  `fromstringlist`, `XML`, and via `XMLParser`), str **and** bytes / file inputs;
- **same-raise:** raises iff defusedxml raises, with the matching purexml exception
  (`DTDForbidden`/`EntitiesForbidden`/`ExternalReferenceForbidden`, or stdlib
  `ParseError` for malformed);
- **the knob matrix:** all 8 `forbid_dtd`×`forbid_entities`×`forbid_external`
  combinations must match defusedxml — *including the deliberately-unsafe combos*
  (e.g. `forbid_entities=False` allows entity expansion). Equivalence means
  matching the oracle's choices, not "always safest."

## 5. Acceptance bar (binding, falsify-first)
1. **C14N same-parse equivalence vs `defusedxml.ElementTree`** for `parse` (file +
   file-like), `fromstringlist`, `XML`, and the exposed `XMLParser` — over the
   v0.1 corpus + file inputs — **0 disagreements.**
2. **Attack battery per entry point** — the 7-class corpus run through `parse`/
   `fromstringlist`/`XMLParser`, each blocked exactly as defusedxml blocks it, with
   the no-fetch/no-read proof harness. (A file-fed XXE is the key new case.)
3. **`forbid_*` knob-matrix equivalence** — all 8 combinations, vs defusedxml, on
   both an allow-doc and each attack class. Confirms `DTDForbidden` fires iff
   `forbid_dtd=True` and matches defusedxml's `DTDForbidden`.
4. **No new runtime deps; determinism; never-crash** — unchanged from v0.1.

## 6. Measure-first (first task)
Read `defusedxml.ElementTree`'s `parse`/`fromstringlist`/`XML`/`XMLParser` +
`common._generate_etree_functions` to confirm exact signatures, the `ElementTree`-
vs-`Element` return types, and `DTDForbidden`'s shape; enumerate the knob-matrix
behaviors against the oracle before trusting the implementation. (Most of the
engine work is done; this slice is breadth + the `forbid_dtd` path + the namespace.)

## 7. Out of scope for v0.2.0
- **`iterparse`** → v0.3 (the hard, streaming one).
- **minidom / sax / pulldom / xmlrpc / lxml** → post-1.0 (deferred, not excluded).
- **Version-assertion + differential-fuzzing durability** → the v0.1.2 patch.
- **Adoption model / packaging / PyPI / license** → deferred to 1.0.

## 8. Definition of done
- `parse`, `fromstringlist`, `XML`, `tostring`, exposed `XMLParser`, and the
  `forbid_*` knobs (+ `DTDForbidden`) implemented under `purexml.ElementTree`,
  stdlib-only, deterministic.
- Acceptance bar §5 green (C14N sweep + per-entry-point attack battery + knob matrix).
- Measure-first notes recorded; `HISTORY.md` entry; four-leg review.
- `COMPLIANCE-v0.2.md` written at the before-merge gate.
