# purexml v0.3.0 RFC — hardened `iterparse` (complete the ElementTree family)

> **Status: Approved** (2026-06-16, by Russell; design fork resolved — Option A).
> Closes out the ElementTree family per `docs/ROADMAP-to-1.0.md` (G3, v0.3). Stated
> as capabilities + acceptance + the design decision, not implementation —
> implementation may proceed, with a `docs/COMPLIANCE-v0.3.md` due before merge.
> **This is the one genuinely risk-bearing slice** — incremental hardening under
> streaming.

## 1. Scope of this slice
Add **`iterparse`** — incremental, event-driven parsing — as the last missing piece
of the `defusedxml.ElementTree` surface. After v0.3, `purexml.ElementTree` mirrors
defusedxml's full ElementTree surface (the v0.2 "iterparse absent → AttributeError"
honesty note is resolved), and purexml is **1.0-freeze-ready** on the build axis
(pending adoption validation + the deferred decisions + the freeze ceremony).

## 2. Capability delivered
`iterparse(source, events=None, parser=None, forbid_dtd=False, forbid_entities=True,
forbid_external=True)` under `purexml.ElementTree`, behaviorally equivalent to
`defusedxml.ElementTree.iterparse`:
- returns an iterator of `(event, elem)` pairs; events `"start"`, `"end"`,
  `"start-ns"`, `"end-ns"` (default: `"end"` only), plus `"comment"`/`"pi"` where
  stdlib supports them;
- parses incrementally from a filename or file-like `source` (opening/closing a
  filename source, like stdlib);
- blocks the same attack classes **during streaming**, raising the same purexml
  exception at the same point in the stream defusedxml raises.

## 3. Design decision — wiring hardened parsing into incremental events (the crux)
Stdlib `iterparse` wraps the parser in `XMLPullParser`, which drives it via the
**private** `parser._setevents(events_queue, events_to_report)` API
(`xml/etree/ElementTree.py`). defusedxml gets this *for free* by subclassing the
pure-Python `XMLParser` (it inherits `_setevents`). purexml's hand-built `XMLParser`
(on `xml.parsers.expat`) does **not** have `_setevents`. Two options:

- **Option A (recommended) — implement `_setevents` on `purexml.XMLParser`, reuse
  stdlib `iterparse`.** Mirror the stdlib `_setevents` (it wires expat
  start/end/start-ns/end-ns/comment/pi handlers to append `(event, data)` to the
  queue), then `purexml.ElementTree.iterparse = lambda source, events=None,
  parser=None, **knobs: _stdlib_iterparse(source, events, parser or XMLParser(**knobs))`.
  This reuses stdlib's proven `iterparse`/`XMLPullParser`/iterator + `close_source`
  semantics — *exactly defusedxml's effective approach.* New code is small (the
  `_setevents` mirror + comment/pi event wiring). **Risk:** `_setevents` is a private
  API; mitigated by the CI matrix (3.10–3.13) + the event-stream equivalence sweep
  catching any drift.
- **Option B — purexml's own iterator.** Reimplement the chunked-read + event-queue
  + `close_source` machinery. No stdlib-internal coupling, but re-derives proven
  code (iterator `__del__`/close semantics, root return) — more surface to get wrong.

**DECISION (2026-06-16, Russell): Option A for v0.3** — lowest equivalence risk
(reuses the same machinery defusedxml does), smallest new surface; the private-API
coupling is a *known, matrix-tested* cost, preferable to re-deriving the iterator.

**Option B is NOT pursued in purexml — it is marked as a possible separate project.**
An independent, stdlib-internals-free hardened streaming parser is a materially
different thing from purexml's "faithful minimal-surface defusedxml mirror" charter
(more code, more audit burden, not a 1:1 mirror), so it deserves its own identity
rather than bloating purexml's freeze. Recorded in `scratch/spinoff_ideas.md`;
revisit post-1.0 if there's real demand.

## 4. The streaming-security bar (what makes this slice hard)
- **No-fetch under streaming.** The blocking handlers (`EntityDecl`,
  `ExternalEntityRef`, and `StartDoctypeDecl` when `forbid_dtd`) fire **during
  `feed`**, i.e. as chunks arrive — before the consumer iterates past them. So
  bombs/XXE are blocked at the declaration mid-stream. MUST be proven with the
  no-I/O trip-wire **while consuming the iterator**, not just on a one-shot parse.
- **Reparse-deferral interaction (CVE-2023-52425).** Chunked `feed` + Expat 2.6.0+
  reparse deferral means a token (or a malicious large token) may span feeds and be
  deferred. Verify the blocking handlers still fire when the construct completes, and
  that large-token behavior matches defusedxml (version-gated on `pyexpat.EXPAT_VERSION`).
  Account for the 3.13 `flush()` semantics.
- **Cleanup-on-error through the iterator.** When iteration raises mid-stream
  (blocked/malformed), the heavy parser + partial tree must be released (the v0.1.2 /
  v0.2 reference-cycle concern, now reached *through* the pull-parser/iterator and
  `close_source`). Prove the heavy state is freed and a filename source is closed.
- **Source handling.** Filename (purexml/stdlib opens + closes it) vs file-like
  (caller owns it) — match stdlib's `close_source` behavior exactly.

## 5. Acceptance bar (binding, falsify-first)
1. **Event-stream equivalence vs `defusedxml.iterparse`.** For each events mode
   (`None`, `("start","end")`, the full set incl. start-ns/end-ns/comment/pi), the
   yielded sequence of `(event, C14N(elem))` MUST match defusedxml's exactly, over
   the corpus, for filename and file-like sources — **0 disagreements.**
2. **Same raise-point.** For each attack class, consuming `purexml.iterparse` MUST
   raise a `PureXMLError` (or stdlib `ParseError` for malformed) at the same stream
   position defusedxml raises — proven by iterating both and comparing where each stops.
3. **No-fetch/no-read proven under streaming** — the socket/open/urlopen trip-wire
   active while the iterator is consumed; sentinel never read.
4. **`forbid_*` knob matrix for `iterparse`** — all 8 combos match defusedxml.
5. **Reparse-deferral / large-token equivalence** — version-gated.
6. **Cleanup-on-error** — a blocked/malformed stream releases the heavy parser state
   and closes a filename source.

## 6. Measure-first (first task)
Read stdlib `iterparse` / `XMLPullParser` / `_setevents` / the iterator + `close_source`
machinery and defusedxml's `iterparse`; confirm the `_setevents` contract, the exact
event set, and the iterator close semantics **across 3.10–3.13** (it is a private API —
pin what each version expects). Record the enumerated contract.

## 7. Out of scope for v0.3.0
- `minidom` / `sax` / `pulldom` / `expatreader` / `expatbuilder` / `xmlrpc` / `lxml`
  — post-1.0 (deferred, not excluded). **This RFC completes the ElementTree family;
  it is the last build slice before the 1.0 freeze gate.**

## 8. Definition of done

- Acceptance bar §5 green (event-stream + raise-point equivalence, no-fetch-under-

- Measure-first `_setevents` contract recorded; `HISTORY.md` entry; four-leg review —
  **extra adversarial weight on the streaming paths.**
